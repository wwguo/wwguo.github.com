第一篇¶

Date:2006-11-26
Time:23:43
有些事情确实不应该强求，其本质决定了效率。文字编辑器需要付出很多，才能达到图形工具的基本效果。没有必要把自己陷在自以为高深的东西里，需要反省……

记录了这么多，最后还是得放掉，这个笔记就留在这里吧。

Etask Mode 是个被丢弃的项目，所以很多方面还不完全。最初是从 Gantt 图查到 Etask，原因只是不想安装 Gnome 的 Planner，前者是靠文字显示后者用图形显示的东西。如果是用作个人管理，Planner、Org-mode 都比 Etask 强的多。所以说，Etask 真的是个鸡肋，唯一的好处是让我温习了项目管理中的一些要点。

Etask 安裝¶

因为 Etask 中有多个文件，所以需要设置 load-path，然后 require，而非 load-file。如我把 Etask 放在 ~/.emacs.d/ 中

(add-to-list 'load-path "~/.emacs.d/etask")
(require 'etask)

依赖的 Emacs 扩展包括：calendar、timezone、cal-iso 和 highlight-current-line（可选）

如果想利用 LaTeX 文件生成高质量的 Gantt 图，则依赖的 LaTex 宏包括：ae、babel、inputenc、fontenc、eurosym、amssymb、pst-all 和 fancyhdr

Emacs 中撤销任务用命令 C-x u，撤销 n 个任务用命令 C-u n C-x u。

设置任务集合¶

M-x etask 启动 Etask Mode，会有 3 个选择 (1) Project；(2) Todo；(3) Event。

Project¶

选择 1 定义一个项目，首先输入项目名称，然后输入项目任务的数目，之后就需要一个个地输入项目任务的名称了，输入名称时可以使用 TAB 键，Etask 会提供一些名称建议，不过不用太担心，以后可以再定制，简单的修改 tasklist 即可。

需要注意：

  • 项目名称中的空格需要用 M-SPC 输入，而非单纯用 SPC；
  • 变量 case-fold-search 控制是否大小写敏感，首部的空格会被忽略；

进入项目任务管理界面后，可以看到很多的 M。M 在 Gantt 图中的含义是该行的任务中实际上没有任务，只是个 milestone（即需要的精力是 0）。

插入新任务用 i，C-u i 插入子任务。用命令 C-u n X 可以设置显示的子任务的层级，其中 n 为层级数。

分配精力资源¶

命令 1，光标就会转到任务 1，命令 m 标记任务、u 取消标记，命令 M 标记所有任务。

为所有标记的任务计划精力使用命令 e，为每个任务设置任务时期（task duration），w 表示周、d 表示天、h 表示小时，如 2w 表示两周。如要定义 milestone，则将计划精力设置为 0，0w、0d、0h 都可以。输入的任务时期可是小数。

然后再标记所有任务，用命令 f 分配所需的人力，单位为 FTE (full-time equivalents)。

然后再用 z a 确定 Gantt 图显示了所有任务。

注意 1 个人在 1 个任务上全时工作时的 FTE 为 1.0，如果 1 个花 30% 的时间在任务上，则他在该项目任务上的负载就是 0.3 FTE。因此需要所有人在该任务上的 FTE，比如一个任务的计划精力是 20 天，你输入 0.5，则完成该任务的工作时间要求就是 40。

有时任务条会比期望的长，查查看是任务开始和结束的时期中否其中有周末或假期，可以用 M-x calendar 调用日历查看。Etask 可以用 calendar day 或 business day 计算时间，用命令 C-c C-d 检查 Etask 的相关行为，用 C-c d 切换。Etask 用变量 calendar-holidays 和变量 etask-holiday-regexp 识别假期，确保其配置正确。

需要注意，Emacs 中的 Gantt 图只能精确到光标长度，这是 text mode 的局限造成的，所以有时稍有差异的时期会显示为一样。

识别关键任务¶

关键路径（critical path）是指超出预期时间就会对整个项目进度产生影响的任务。

用命令 p、C-p 或 UP，用命令 n、C-n 或 DOWN 键移动光标到任务行。

可以先标记关键任务路径，然后用命令 c 设置关键任务。关键路径任务的任务名颜色会改变，任务状态也会包含“critical”字样。颜色可以设置。取消关键路径任务设置只需再用 c 命令。

如想改变任务的顺序，则只需在任务行上用命令 C-M-u（移上）或 C-M-d（移下），如要删除一个项目，则用删除或粘贴命令 C-k 或 C-y。

识别风险任务¶

不在关键路径内，但你认为是重要的或有风险的任务，用命令 h 设置。同样，任务命令颜色会改变，任务状态也会改变。再用 h 命令可以取消风险任务标记。

规划任务进度¶

有 3 种设置任务开始时间的方法：

Link Tasks¶

Link Task 是规划新任务时推荐使用的方法，其他两种方法适合调整或推延进度。Link Task 是指连接所有关键路径任务。

连接所有关键路径任务用命令 l，然后用命令 x，最后 Etask 会要求你输入相对开始日期，0 表示前一任务结束时，后一任务就开始；3 表示当前任务是在前一任务开始后第 3 天开始；-2 表示前一任务结束前 2 天预先开始后一任务。如果任务不能完全显示，则用命令 z a。

如果要改变关键路径任务的顺序则需要用命令 C-M-u 或 C-M-d。设置项目开始的时间用命令 b ，如果不能设置起始时间，则输入 C-c d，直到 Etask Mode 回答 clander days，或者也可以直接配置 Etask Mode。

Move Tasks¶

用 C-c 和 b（begin data）、d（due data）、f（forward）和 p（postpone）组合修正进度，如 C-c f b 表示提前任务的开始时间； C-c p b 表示推迟任务开始时间等等。可以用 C-c p t 设置推迟的时间，t 为时间。
